# Капсула (силуэты/сочетания) как эталон для AI-генерации образов

Дата: 2026-06-22
Статус: на согласовании

## Проблема

Кураторская «капсула» (55 базовых вещей с силуэтами + 46 сочетаний Casual/Classic/Evening),
которую загружали вручную, лежит мёртвым грузом: читается/пишется только админкой
(`backend/app/api/basic_items.py`), а генерация образов (cron, чат, gap-fill, виджет) её
не использует — Gemini выдумывает сочетания «с нуля» по названиям вещей. Цель: кормить
капсулу нейросети, чтобы образы повторяли кураторскую логику.

## Реальность данных (проверено в проде 2026-06-22)

- `basic_wardrobe_items`: 55 строк, `gender` заполнен, силуэт — в `description` («Джинсы Baggy», «Брюки прямые»). Колонки `color` НЕТ.
- `combinations`: `name` = повод (Casual/Classic/Evening), `combination_type='items'`, `description` пуст. Колонок `gender`/`style` в проде НЕТ (локальный CRUD-код их пишет → дрейф схемы, см. «Вне охвата»).
- `combination_elements` связывает сочетание с базовыми вещами; пол сочетания выводим из вещей-членов.
- Цвета структурно в БД отсутствуют → «цвета» = палитра `lib/color-map.ts` (фронт). В промпт идёт статичный мини-блок цветовой гармонии, не данные.
- AI-чат: `backend/app/api/misc.py`; gap-fill: `backend/app/api/recommendations.py`; cron: `backend/app/api/cron.py`; виджет: `backend/app/api/widget.py` (только CLIP `/clip/complement`, без LLM).

## Подход

Few-shot паттерны в промпт (подход A). Сочетания, подходящие по полу, рендерятся в текстовые
эталоны; модель имитирует их на реальных вещах. Ноль изменений схемы.

Отвергнуто: B (retrieval-anchored, жёсткие слоты) — хрупко, кода больше; C (движок правил
силуэт+цвет) — требует отсутствующих цветовых данных, YAGNI.

## Компоненты

### 1. `backend/app/services/capsule.py` (новый, единственный новый файл)
- `capsule_style_guide(gender: str) -> str`
  - Один SQL: `combinations ⋈ combination_elements ⋈ basic_wardrobe_items`,
    члены фильтруются по `gender` (+ NULL/unisex).
  - Каждое сочетание → строка-паттерн: `«{повод}: {силуэт+тип} + {силуэт+тип} + …»`.
  - Дописывает статичный блок цветовой гармонии (нейтральная база + 1 акцент, без кричащих сочетаний).
  - `@lru_cache` по `gender` — данные капсулы почти не меняются.
    `ponytail: lru_cache; добавить cache_clear() при правке сочетаний, если станет проблемой.`
  - Пустая капсула / ошибка БД → возвращает `""` (промпты работают как раньше, без падения).

### 2. Точки вставки (по одной f-строке каждая)
- cron `_gemini_organize` — `backend/app/api/cron.py:165` (блок перед «ОБЯЗАТЕЛЬНЫЕ ПРАВИЛА»).
- чат — `backend/app/api/misc.py` (в system/user-промпт).
- gap-fill — `backend/app/api/recommendations.py` (в промпт подбора).

### 3. Виджет — Gemini-полировка на cache-miss
- В `backend/app/api/widget.py` после CLIP `/clip/complement` и до записи в `widget_reco_cache`:
  один Gemini-вызов с тем же капсульным блоком — переименовать образы стильно и при необходимости
  подрезать/переупорядочить под эталонный паттерн. Состав ограничен ID, которые вернул CLIP
  (не добавляем новых вещей).
- Платит только первый просмотр корзины (кэш 30 мин). При ошибке/таймауте Gemini → отдаём
  CLIP-выдачу как есть (graceful fallback).

## Поток данных

`gender → capsule_style_guide() → текстовый блок → склейка в prompt → Gemini → образы из реальных ID`

Виджет: `CLIP /complement → Gemini-полировка (тот же блок) → кэш → образы`.

## Обработка ошибок

- `capsule_style_guide` никогда не бросает наружу: при пустых данных/ошибке БД → `""`.
- Виджет: ошибка/таймаут Gemini → fallback на сырую CLIP-выдачу.
- Существующие fallback'и Gemini (cron/chat/gap-fill) сохраняются.

## Проверка

- `backend/app/services/capsule.py` — self-check `__main__`/assert: на фикстуре из 1 сочетания
  (2-3 вещи) функция возвращает строку с поводом и силуэтами; на пустом входе → `""`.
- Ручная: прогнать cron-генерацию на тест-юзере, глазами сверить, что образы повторяют
  кураторские паттерны и не смешивают пол.

## Вне охвата (флагнуть отдельно)

- Дрейф схемы: локальный CRUD `combinations` пишет `style`/`gender`, которых нет в проде →
  админка сочетаний, вероятно, падает на вставке. Чинится отдельной миграцией; здесь пол
  выводим из вещей-членов, миграция не нужна.
- Структурные цветовые правила на уровне вещи/сочетания (потребуют новых данных).
